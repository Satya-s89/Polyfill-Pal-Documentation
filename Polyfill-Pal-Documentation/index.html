Getting Started

eslint-plugin-baseline is an ESLint plugin that helps you write cross-browser compatible JavaScript. It detects the use of web features that do not have "Widely available" baseline support according to the official web-features dataset. This guide will walk you through installing the plugin and configuring your project to use it.

Installation

First, ensure you have ESLint installed. Then, add the eslint-plugin-baseline package to your project's devDependencies by running the following command in your terminal: 

npm install https://github.com/Satya-s89/Polyfill-Pal/raw/main/eslint-plugin-baseline/eslint-plugin-web-baseline-1.0.0.tgz --save-dev

Configuration

The plugin uses the new ESLint flat configuration format. Create an eslint.config.js file in the root of your project or add the following configuration to your existing file.

ESLint Flat Configuration (eslint.config.js)

import baselinePlugin from 'eslint-plugin-baseline';
import js from '@eslint/js';

export default [
  js.configs.recommended,
  {
    files: ["**/*.js"],
    plugins: {
      "baseline": baselinePlugin
    },
    languageOptions: {
      ecmaVersion: 2023,
      sourceType: "module",
    },
    rules: {
      'baseline/no-non-baseline-features': 'error'
    }
  }
];

Legacy Configuration (.eslintrc.js)

For projects still using the legacy configuration, you can configure the plugin like this: 

module.exports = {
  plugins: ['baseline'],
  rules: {
    'baseline/no-non-baseline-features': 'error'
  }
};

Rules

baseline/no-non-baseline-features

This rule disallows the use of specific JavaScript methods and features that do not have "Widely available" baseline support. When this rule is triggered, the linter will report an error with a message that includes the current baseline status.

How it works:
The plugin analyzes your code for methods that are part of a custom feature map. It then checks the corresponding feature's status in the web-features dataset. If the status is not "high", it reports an error.

Code Examples

✅ Valid Code (Passes Linter)

// These features have "high" baseline support (widely available)
const obj = { a: 1, b: 2 };
const keys = Object.keys(obj);

const str = 'hello';
const padded = str.padStart(10, '0'); // Widely supported

❌ Invalid Code (Triggers Error)

// These features have "low" baseline support (limited availability)
const arr = [3, 1, 2];
const sorted = arr.toSorted(); // ❌ This will trigger an error

const copy = arr.toReversed(); // ❌ This will trigger an error

Description

POLYFILL-PAL is a comprehensive ESLint plugin designed to help developers write cross-browser compatible JavaScript code. By leveraging the official web-features dataset, it identifies and flags the use of JavaScript methods and APIs that lack widespread browser support, ensuring your code works reliably across different environments.
